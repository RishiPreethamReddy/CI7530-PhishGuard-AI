# Project Management Documentation

## Project Aim
The aim of this project is to design and develop an AI-assisted phishing email risk assessment artefact that supports early-stage cyber-security analysis.

## Project Plan

| Stage | Activity | Deliverable |
|---|---|---|
| Stage 1 | Review coursework brief and select artefact | PhishGuard AI idea selected |
| Stage 2 | Define aim, objectives, users and scope | Project scope completed |
| Stage 3 | Design artefact workflow | Input-output workflow created |
| Stage 4 | Develop Python Streamlit app | Working prototype |
| Stage 5 | Test using sample emails | Testing evidence table and screenshots |
| Stage 6 | Prepare GitHub repository | Code and documentation uploaded |
| Stage 7 | Prepare handover notes | Viva/handover explanation completed |

## Project Log

| Day | Work Completed | Reflection |
|---|---|---|
| Day 1 | Selected phishing email risk assessment as the artefact topic | The topic is realistic and strongly linked to cyber-security |
| Day 2 | Planned the artefact features and risk scoring method | A transparent scoring method is easier to explain during handover |
| Day 3 | Built the first Streamlit prototype | The tool can now accept email input and generate a result |
| Day 4 | Added explanation and recommended action output | Explanation makes the tool more useful than a simple score |
| Day 5 | Created sample test emails | Testing evidence supports the artefact |
| Day 6 | Prepared documentation and README | The project is easier to hand over |
| Day 7 | Final review and GitHub upload | The artefact is ready for assessment review |
